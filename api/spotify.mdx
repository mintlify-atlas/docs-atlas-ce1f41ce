## Overview

The Spotify service provides server-side integration with the Spotify Web API using OAuth2 client credentials flow. It handles authentication token management with automatic caching and expiration, and provides methods to retrieve artist top tracks.

**Source Location**: `src/server/services/spotify/`

<Note>
  This service is designed to run exclusively on the server. Never expose authentication credentials to the client.
</Note>

## Authentication

### getSpotifyToken()

Retrieves a valid Spotify access token using client credentials flow. Checks cache before requesting a new token to optimize API usage.

**Location**: `src/server/services/spotify/auth.ts:15`

```typescript
async function getSpotifyToken(): Promise<string>
```

#### Returns

<ResponseField name="access_token" type="string" required>
  Valid Spotify access token for API requests
</ResponseField>

#### Authentication Flow

1. **Cache Check**: Verifies if a valid token exists in cache
2. **Token Request**: If cache is empty/expired, requests new token from Spotify
3. **Authorization**: Uses Basic Auth with Base64-encoded `client_id:client_secret`
4. **Caching**: Stores token in cache with 60-second buffer before expiration

#### API Endpoint

```
POST https://accounts.spotify.com/api/token
```

#### Request Headers

<ParamField header="Content-Type" type="string" required>
  `application/x-www-form-urlencoded`
</ParamField>

<ParamField header="Authorization" type="string" required>
  `Basic {base64(client_id:client_secret)}`
</ParamField>

#### Request Body

<ParamField body="grant_type" type="string" required>
  Must be `client_credentials`
</ParamField>

#### Response Type: SpotifyToken

```typescript
interface SpotifyToken {
  access_token: string;    // OAuth2 access token
  token_type: string;      // "Bearer"
  expires_in: number;      // Token lifetime in seconds
}
```

#### Error Handling

```typescript
try {
  const token = await getSpotifyToken();
} catch (error) {
  // Throws: "Fallo en la autenticación de Spotify"
}
```

<Warning>
  Requires `SPOTIFY_CLIENT_ID` and `SPOTIFY_CLIENT_SECRET` environment variables.
</Warning>

## Track Methods

### getArtistTopTracks()

Retrieves the most popular tracks for a configured artist. Results are cached for 1 hour to reduce API calls.

**Location**: `src/server/services/spotify/tracks.ts:14`

```typescript
async function getArtistTopTracks(): Promise<Track[] | null>
```

#### Parameters

No parameters required. Uses `SPOTIFY_ARTIST_ID` from environment configuration.

#### Returns

<ResponseField name="tracks" type="Track[] | null">
  Array of track objects, or `null` if error occurs or artist ID not configured
</ResponseField>

#### Response Type: Track

```typescript
interface Track {
```

#### API Endpoint

```
GET https://api.spotify.com/v1/artists/{artist_id}/top-tracks?market=ES
```

#### Request Headers

<ParamField header="Authorization" type="string" required>
  `Bearer {access_token}`
</ParamField>

#### Example Usage

```typescript
import { getArtistTopTracks } from '@/server/services/spotify/tracks';

const tracks = await getArtistTopTracks();
if (tracks) {
  tracks.forEach(track => {
    console.log(`${track.name} - ${track.artists[0].name}`);
    console.log(`Popularity: ${track.popularity}`);
    console.log(`Duration: ${track.duration_ms}ms`);
  });
}
```

#### Error Handling

Returns `null` on errors:
- Missing `SPOTIFY_ARTIST_ID` environment variable
- Authentication failures
- Network errors
- Invalid API responses

```typescript
const tracks = await getArtistTopTracks();
if (!tracks) {
  console.error('Failed to retrieve tracks');
}
```

## Error Messages

| Error | Cause | Solution |
|-------|-------|----------|
| "Fallo en la autenticación de Spotify" | Token request failed | Verify client credentials and network connectivity |
| "No se ha definido un SPOTIFY_ARTIST_ID" | Missing artist ID configuration | Set `SPOTIFY_ARTIST_ID` environment variable |
